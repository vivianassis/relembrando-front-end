Campos de busca

TAG "form"

Na TAG < form >, que não possui representação na interface, mas "abraça" os campos, existem atributos específicos que englobam todo o formulário:
Vale ressaltar aqui também, uma tag muito importante, que também é usada em praticamente todo o formulário, a tag < label >.
Ela serve para dar atributos as descrições que explicam os campos(por exemplo, nome, idade, gênero, etc) e não tem uma representação gráfica dentro do formulário.

  • name: o nome do formulário
  • method: qual dos metodos HTTP irá ser usado
    • Melhor POST nesse caso, pois não usa a url com os dados do formulário, mas também pode ser GET
  • action: endereço do servidor (ou site) onde será enviado o formulário
  • target: abre na mesma aba ou não, após o envio
  • autocomplete: Se completar automaticamente baseado no cache (gravar a senha por exemplo)
  • "on[...]": Geralmente usando o 'onsubmit' para formulários

TAGS "input"

Input é um campo onde inserimos valores. É importante conhecer o navegador ao estudar input, pois essa TAG é dependente de compatibilidade para a exibição de certos campos.

Types de input:

  • Text: Aceita todo tipo de entrada(caractere especial, número, e texto

  • Number: Aceita apenas números (e a letra e, número científico)

    Exemplos de atributos da tag number (min e max podem ser alterados entre si, invertendo os significados)
    - min = Número mínimo aceito
    - minlength = Tamanho mínimo do preenchimento (em caracteres)
    - step = Intervalo de pulos que da ao usar as setas do input

  • Button: Cria um botão clicável.

    • Características do button:
    < button type =
    - button [clicável]
    - menu [abre um submenu]
    - reset [limpa os campos]
    - submit [envia o formulário]
    > "NOME" < /button >

  • Range: Cria um campo scroll, que aceita os mesmos types do campo number (min, max, etc)
    No campo range podemos carregar o campo já num determinado valor, usando a tag value e setando o valor desejado, que será o inicialmente carregado no scroll.

  • Color: Cria um seletor de cor.

  • Email e URL: cria um campo com validações predefinidas(com @ para e-mail, ou com http para url, por exemplo).

  • Date: Cria conforme o navegador usado, um campo para inserir data.

  • Checkbox: Usado geralmente como valor booleano, cria um campo para selecionar (ou não, uma opção)

  • Radio: Usado da mesma maneira do checkbox, porém definindo um 'name' em comum, ele impede que se possa selecionar duas opções ao mesmo tempo.

  • Tel: Cria um campo com a formatação editável e/ou predefinida para telefone. Aparentemente é um campo de texto comum.
    Isso ocorre pois outros paises tem diferentes tipos de número de telefone, incluindo diversos tipos de caracteres.
    Muitos navegadores atuais ainda não implementaram a tag 'tel' ainda.

  • Hidden: Campo invisível ao usuário do navegador, mas que permanece enviando dados ao servidor.

  • File: Campo para envio de arquivos. Nesse campo pode colocar um novo atributo 'sem valor atribuído', chamado /multiple.
    Isso pode ser feito desta forma: < input type="file" multiple/ >

    • Extra!

    A partir do HTML 5 muitos dos campos acima foram implementados no Google Chrome.
    Um dos que prometem ser bastante uteis na criação, não exatamente só dos formulários mas também isoladamente é o campo 'search'.
    Como o próprio nome já diz, trata-se de um campo usado para buscar apresentando diversas funcionalidades dentro dos sites atuais, como os e-commerces. O diferencial desse campo é sua função de limpar a 'busca', com um clique.
